### Task 6: Add stall detection in cycle transitions to orchestrator
Add stall detection to `commands/minion.md` Step 7 phase progression.

Two changes:
1. In **Case B** (cycle target completed with non-success, e.g. review_failed), BEFORE spawning the fix worker: capture `pre_fix_sha = git rev-parse {branch_name}` and store in task tracking metadata.

### Task 7: Update orchestrator Step 1.7 for scored agent matching
Replace the hardcoded auto-detection rules in `commands/minion.md` Step 1.7 with scored matching that reads the `matches` field from agent YAML frontmatter.

Changes to "Parsing Agent Files" section: add `matches` to the list of extracted fields (object with extensions, paths, keywords arrays).

Replace "Auto-Detection Rules" section entirely with scored matching:
- Extension match = 3 points per matching file
- Path match = 2 points per matching file
- Keyword match = 1 point per keyword found in task description
- Highest score wins (minimum 1 to qualify)
- Ties: project-level preferred, then alphabetical
- Score 0 for all → minion-worker fallback
- Agents without `matches` field excluded from scored matching (can still be assigned via explicit Agent: field)
